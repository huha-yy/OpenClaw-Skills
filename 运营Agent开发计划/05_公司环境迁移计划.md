# 阶段 4：公司环境迁移计划

## 1. 阶段目标

本阶段目标是将本地 POC 迁移到公司测试环境，验证稳定性、安全性和可维护性。

## 2. 迁移前提

迁移前需要满足：

```text
1. 本地 POC 已跑通。
2. 运营认可发布包输出格式。
3. CTO 确认第一版范围。
4. 公司关键词和品牌规范已整理。
5. 模型调用方式和密钥管理方式已确认。
```

## 3. 本地与公司环境差异

| 本地 POC | 公司环境 |
| --- | --- |
| 本地文件夹 | 对象存储或共享存储 |
| 本地配置文件 | 配置中心或数据库 |
| 本地脚本 | 任务调度系统 |
| 本地日志 | 集中日志 |
| 用户手动查看文件 | 审核页面或运营工作台 |
| 本地环境变量 | 密钥管理系统 |

## 4. 公司环境需要补充的能力

```text
1. 权限管理
2. 审核流
3. 任务日志
4. 失败重试
5. 发布包版本管理
6. 密钥管理
7. 多品牌/多账号隔离
8. 运行监控和告警
```

## 5. 推荐迁移顺序

```text
1. 先迁移配置和文件输出。
2. 再迁移任务调度。
3. 再接入集中日志。
4. 再做审核页面。
5. 最后考虑平台接口。
```

## 6. 不建议公司环境第一版直接做

```text
1. 不建议直接接抖音、小红书自动发布。
2. 不建议直接做自动点赞、评论、转发。
3. 不建议在没有审核流的情况下自动发布热点内容。
4. 不建议跳过日志和审计。
```

## 7. 本阶段交付物

- 公司环境部署方案
- 配置迁移方案
- 存储迁移方案
- 任务调度方案
- 密钥管理方案
- 审核和权限方案
