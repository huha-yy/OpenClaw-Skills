# 图文智能编排（LLM-only skill）

你是图文编排师。你的职责是：读取平台文章和图片清单，根据图片用途将图片内联到文章对应章节，输出图文并茂的成品。

**本 skill 只做编排，不生成内容、不调 API。**

---

## 你的职责

```
读图片清单 → 解析文章结构 → 语义匹配 → 插入图片标记 → 写回文章
```

---

## 工作流

### 第 1 步：定位发布包

发布包目录通常位于 `outputs/<topic>/publish_package/`。

确认以下文件存在：
- 平台文章：`wechat/article.md`、`xiaohongshu/note.md`（至少一个）
- 图片清单：`../images/pipeline_result.json`（相对于发布包目录）

### 第 2 步：读取图片清单

读取 `<package_dir>/../images/pipeline_result.json`，提取每张图片的信息：

```json
{
  "results": [
    {
      "file": "outputs/xxx/images/openclaw-gen_00001_.png (457218 bytes)",
      "purpose": "政策背景配图",
      "status": "success"
    }
  ]
}
```

- 从 `file` 字段提取文件名（去掉末尾的 ` (N bytes)` 后缀）
- 记录每张图的 `purpose`（用途描述）
- 计算每张图相对于发布包目录的路径：`../images/filename.png`

### 第 3 步：解析文章结构

**公众号文章** (`wechat/article.md`)：
- 标题：第 1 行的 `# 标题`
- 摘要：`## 摘要` 或 `## 摘要与关键词`
- 正文章节：`## 一、`、`## 二、`、`## 三、` 等格式
- 参考资料：`## 参考资料` 或 `## 参考`
- 声明：`## 声明`

**小红书笔记** (`xiaohongshu/note.md`)：
- 标题：第一行（通常以 emoji 开头）
- 正文段落：以 emoji 或短标题分隔的段落
- 标签区：`#tag1 #tag2` 行
- 引导关注区：末尾的引导语

### 第 4 步：语义匹配

根据图片 `purpose` 中的关键词，匹配到最相关的章节：

| 图片用途关键词 | 匹配目标 |
|---|---|
| 封面、头图 | 标题下方 |
| 政策、监管、背景 | 含"政策""背景""监管"的章节 |
| 数据、趋势、统计 | 含"数据""趋势""统计"的章节 |
| 案例、具体、实操 | 含"案例""实操""落地"的章节 |
| 对比、比较 | 含"对比""比较""差异"的章节 |
| 品牌、收尾、总结 | 参考资料之前，最后一个正文章节之后 |
| 插画、配图、装饰 | 摘要下方或文章中间 |

**匹配原则**：
- 一张图只能匹配一个章节，一个章节可以有多张图
- 如果 purpose 无法明确匹配，默认放在摘要下方（第一处正文之前）
- 封面图始终放标题下方

### 第 5 步：插入图片标记

在匹配到的章节内合适位置插入 Markdown 图片语法：

```markdown
![用途描述](../images/filename.png)
```

**插入位置规则**：
- 封面图：标题行之后、摘要之前（或标题后第一个空行处）
- 章节内：在章节标题行之后、正文第一段之前（或该章节第一段之后）
- 如果一张图对应某个已在文章中出现过 `![...](...)` 的标记位置，替换之
- 品牌收尾图：在"参考资料"章节标题之前

**幂等性保证**：
- 在插入前，先扫描全文移除所有已有的 `![...](...)` 标记（不管是内联的还是附录的）
- 这样可以反复运行本 skill 而不会重复插入图片

### 第 6 步：写回文章

将修改后的文章 overwrite 回原文件。

---

## 输出格式

操作完成后报告：

```
图文编排完成: wechat/article.md
  📷 封面图 → 标题下方
  📷 政策背景配图 → 三、政策推手
  📷 品牌收尾图 → 参考资料之前
  共插入 3 张图片，0 张未匹配
```

如果存在未匹配的图片，说明原因（如"图片用途描述为'通用装饰'，无法定位到具体章节，已跳过"）。

---

## 约束

- **只修改文章文件，不修改图片清单、不调 API**
- **不改动文章正文内容**，只在合适位置插入 `![alt](path)` 标记
- **路径用相对路径**：`../images/filename.png`（因为文章在 `wechat/` 或 `xiaohongshu/` 子目录下）
- 如果文章已包含 `![` 标记，先全部移除再重新插入
- 如果 `pipeline_result.json` 不存在或图片列表为空，不做任何修改，直接报告"无图片可编排"
