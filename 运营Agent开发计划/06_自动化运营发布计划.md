# 阶段 4：自动化运营发布计划

## 1. 阶段定位

自动化运营发布是第二大阶段，建立在第一阶段“热点内容生产 + 发布包 + 审核机制”稳定之后。

第一阶段解决：

```text
内容从哪里来、怎么生成、怎么审核。
```

第二阶段解决：

```text
内容如何进入平台、如何排期发布、如何记录结果、如何处理评论和数据。
```

## 2. 阶段目标

第二阶段目标是从“内容生产辅助”升级到“运营发布自动化”。

核心目标：

```text
1. 支持将发布包转换为平台发布任务。
2. 支持自动创建草稿或发布任务。
3. 支持人工确认后发布。
4. 支持低风险内容自动排期发布。
5. 支持发布状态记录和失败处理。
6. 支持评论收集、分类和回复建议。
7. 支持基础数据回收。
```

## 3. 发布自动化等级

| 等级 | 名称 | 说明 | 是否建议 |
| --- | --- | --- | --- |
| L0 | 发布包导出 | 系统只输出内容包，人工发布 | 已在第一阶段完成 |
| L1 | 自动创建草稿 | 系统创建平台草稿，运营审核 | 第二阶段优先 |
| L2 | 人工确认发布 | 系统准备发布任务，人工确认执行 | 第二阶段核心 |
| L3 | 低风险自动排期发布 | 系统按规则自动发布低风险内容 | 第二阶段后期 |
| L4 | 全自动发布 | 无人确认直接发布所有内容 | 不建议默认开启 |

## 4. 平台接入策略

### 4.1 微信公众号

建议优先接入。

原因：

```text
1. 官方接口能力相对明确。
2. 草稿箱能力适合人工审核。
3. 图文内容结构化程度较高。
```

第二阶段目标：

```text
发布包 → 公众号草稿 → 人工审核 → 发布/群发
```

### 4.2 小红书

建议先保持发布包和人工发布，再评估合规发布能力。

第二阶段目标：

```text
发布包 → 发布任务 → 人工审核 → 人工发布或合规接口发布
```

不建议直接做：

```text
模拟 App 发布
Cookie 登录发布
绕过风控发布
批量自动互动
```

### 4.3 抖音

建议先处理视频内容包、封面、标题、话题，再评估开放平台能力。

第二阶段目标：

```text
视频内容包 → 素材检查 → 发布任务 → 人工确认 → 合规发布能力接入
```

## 5. 发布任务状态机

第二阶段需要引入发布任务状态。

```text
created              已创建
waiting_review       待审核
approved             已通过
rejected             已驳回
draft_created        草稿已创建
scheduled            已排期
publishing           发布中
published            已发布
failed               发布失败
cancelled            已取消
```

每个发布任务需要记录：

```text
任务 ID
热点 ID
平台
账号
内容包路径
审核状态
发布状态
失败原因
发布时间
操作人
日志
```

## 6. 自动化发布工作流

```text
发布包生成
  ↓
平台格式转换
  ↓
风险等级判断
  ↓
审核规则判断
  ↓
生成发布任务
  ↓
创建平台草稿 / 排期任务
  ↓
运营确认
  ↓
执行发布
  ↓
记录发布结果
  ↓
失败重试或人工提醒
```

## 7. 审核规则

自动发布前需要根据风险等级分流。

| 风险等级 | 处理方式 |
| --- | --- |
| low | 可进入自动草稿或自动排期 |
| medium | 必须人工审核后发布 |
| high | 不允许自动发布，只能提醒人工处理 |

高风险内容包括：

```text
公司负面舆情
竞品负面新闻
政策争议
社会争议热点
金融、医疗、法律相关内容
未经证实的信息
```

## 8. 评论与互动自动化

第二阶段可以开始做互动自动化，但建议顺序推进。

优先级：

```text
P1：评论收集
P1：评论分类
P1：高风险评论识别
P1：回复建议生成
P2：人工确认后回复
P2：低风险 FAQ 自动回复
P3：自动点赞、转发、批量评论
```

自动点赞、转发、批量评论不建议作为第二阶段默认能力，应单独评估平台规则、账号风险和业务必要性。

## 9. 数据回收

发布后需要记录基础数据：

```text
发布时间
发布平台
发布账号
阅读量/播放量
点赞量
收藏量
评论量
转发量
涨粉量
私信量
```

第一版数据回收可以先做手动录入或简单导入，后续再接平台接口。

## 10. 技术模块

第二阶段需要新增模块：

```text
publish_tasks/        发布任务管理
platform_connectors/  平台发布 Connector
review_rules/         审核规则
comment_manager/      评论收集与分类
metrics_collector/    数据回收
scheduler/            排期任务
```

## 11. 交付物

第二阶段交付物：

```text
1. 发布任务状态机
2. 平台 Connector 设计
3. 公众号草稿能力
4. 人工确认发布流程
5. 发布结果记录
6. 发布失败重试机制
7. 评论收集和回复建议
8. 基础数据回收
```

## 12. 验收标准

第二阶段可按以下标准验收：

```text
1. 发布包可以转为发布任务。
2. 发布任务有明确状态流转。
3. 公众号内容可以创建草稿或进入待发布状态。
4. 中高风险内容不会自动发布。
5. 低风险内容可以进入排期或人工确认发布。
6. 发布失败可以记录原因并提醒人工。
7. 评论可以被收集、分类并生成回复建议。
```
