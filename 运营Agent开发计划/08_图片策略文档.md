# 图片策略文档

## 1. 文档目的

说明运营 Agent 生成小红书、抖音、微信公众号内容包时，封面图、配图、图文卡片等视觉素材的获取策略。

可选策略两大类：
```text
检索类：Pexels + 高质量图库检索 + 关键词检索（找现成的图）
生成类：Stable Diffusion AI 生图（通过 ComfyUI/A1111/diffusers 等工具调用）
```

## 2. 总体原则

```text
图文协同 > 平台适配 > 品牌一致 > 版权可控 > 审核可追溯
```

第一阶段重点不是自动生成最终图片，而是输出"图文协同方案"和可选素材来源。第二阶段再接入图库 API、AI 生图和自动排版。

## 3. 两大类策略的定位与优先级

### 3.1 策略优先级

| 优先级 | 策略 | 适用场景 | 说明 |
| --- | --- | --- | --- |
| P0 | 图文协同设计方案 | 所有内容 | 先生成每张图的用途、图上文字、视觉说明（必须） |
| P1 | 公司内部素材库 | 产品、品牌、案例 | 最符合品牌，版权最可控 |
| P1 | Pexels 检索 | 生活方式、氛围图、通用场景 | 免费图库，版权清晰，适合泛场景配图 |
| P1 | 高质量图库检索 | 封面、公众号头图、商业质感内容 | 需确认授权范围 |
| P2 | 关键词检索 | 补充素材候选、参考风格 | 仅作候选和参考，不直接用于发布 |
| P2 | Stable Diffusion AI 生图 | 抽象概念图、风格化封面、信息图底图、批量图卡 | 后端模型（SD 1.5/SDXL 等），前端工具可选 ComfyUI（节点工作流/API）、A1111 WebUI（交互式）、diffusers（代码调用） |

### 3.2 三种策略的使用边界

**Pexels + 图库检索（检索类）：**
- 适合：生活方式、办公场景、自然城市氛围、通用商业场景
- 不适合：特定新闻现场、公司/竞品产品图、特定人物肖像、需严格品牌一致的宣传图
- 要求：记录来源、作者、图片 ID、下载链接，方便审计追溯
- Pexels License: https://www.pexels.com/license/

**Stable Diffusion AI 生图（生成类）：**
- 后端模型：Stable Diffusion 1.5 / SDXL / DreamShaper 等开源模型，本地运行
- 前端工具：
  - **ComfyUI**：节点式工作流，可保存复用，支持 API，适合批量标准化生成
  - **AUTOMATIC1111 WebUI**：Web 交互界面，适合调试 prompt 和探索风格
  - **diffusers**：Python 库，适合代码管道集成
- 适合：抽象概念图、风格化封面、信息图背景、场景化插画、社媒卡片底图、批量图卡
- 不适合：真实新闻现场、真实人物肖像、公司/竞品产品图、政策文件截图、高风险事实图
- 要求：保留 prompt + 模型 + 参数、人工审核、确认模型/LoRA 授权范围、不误导为真实现场
- ComfyUI 参考：https://docs.comfy.org/

## 4. 分阶段推荐

### 第一阶段

```text
图文协同方案（必须） + Pexels/图库候选 + AI 生图提示词
```

系统输出：
```text
1. 每张图用途  2. 图上文字  3. 画面描述  4. 对应正文段落
5. 推荐检索关键词  6. 推荐图库来源  7. AI 生图 prompt  8. 风险提示
```

第一阶段不强制自动生成最终图片，ComfyUI 只输出 prompt 和方案不直接跑。

### 第二阶段

```text
图文方案 → 自动检索(Pexels API/公司图库) → 自动评分 → 候选图 → 人工选择 → 发布包

进一步：
图文方案 → ComfyUI 工作流生图 → 自动封面/图卡 → 人工审核 → 发布
```

## 5. 分平台图片策略

| 平台 | 重点 | 优先顺序 |
| --- | --- | --- |
| 小红书 | 图文卡片协同（封面、观点图、步骤图、对比图、总结图） | 图文方案 → 图库检索 → Pexels 氛围图 → AI 生图 |
| 抖音 | 封面 + 视频画面素材 + 字幕卡 | 脚本先行 → 素材库 → 高质量图库 → Pexels → ComfyUI 封面 |
| 公众号 | 封面、头图、正文插图、数据图/流程图、总结卡 | 文章结构先行 → 品牌图/素材 → 高质量图库 → Pexels → AI 概念图 |

## 6. 图片评分规则

| 维度 | 权重 | 说明 |
| --- | --- | --- |
| 相关性 | 30% | 是否贴合热点和内容主题 |
| 平台适配 | 20% | 是否适合目标平台尺寸和风格 |
| 品牌匹配 | 20% | 是否符合公司调性 |
| 版权安全 | 20% | 来源清楚、授权范围明确 |
| 美观度 | 10% | 构图、清晰度、质感 |

候选图输出示例：
```json
{
  "source": "pexels",
  "photo_id": "",
  "photographer": "",
  "url": "",
  "score": 86,
  "reason": "画面符合城市新能源出行主题，适合小红书封面。",
  "risk": "low"
}
```

## 7. 图片生产工作流

```text
内容策略生成 → 图文协同方案 → 生成关键词和 prompt
  → 检索内部素材库 → 检索 Pexels/图库 → 候选图评分
  → 无合适图 → SD/ComfyUI 生图 → 候选
  → 人工审核 → 写入发布包
```

## 8. 风险边界

```text
1. 不用 AI 生成真实新闻现场图
2. 不用 AI 生成真实人物肖像冒充事实图片
3. 不直接复用未经授权的搜索图片
4. 不使用授权不明的模型或 LoRA 做商业发布
5. 不把 AI 生成图伪装成真实拍摄图
6. 不使用竞品商标、产品图做误导性表达
```
